# hantu-kr-bot

한국투자증권 Open API를 활용한 국내 주식 실시간 거래량 돌파 단타 자동매매 시스템.

## 운영 철학

봇 전용 서브 위탁계좌를 개설하여 소액만 입금 후 자산을 격리하여 운영한다.  
100% 롱(Long) 전략만 구동하며, 장 마감 전 전량 청산으로 오버나잇 리스크를 배제한다.  
거래 수수료를 고려해 재진입 쿨다운을 두고, 충동적 추격매매를 구조적으로 방지한다.

***

## 작동 시간

한국 정규장 기준 **09:00 ~ 15:30** (KST)

***

## 핵심 알고리즘

### 매입 조건 (AND)

| 조건 | 기준 |
|---|---|
| 거래량 | KIS API 제공 거래량 폭등 종목 |
| 가격 상승 | 최근 30분간 주가 2% 이상 상승 |
| 재진입 쿨다운 | 청산 후 30분 미경과 종목 제외 |
| 매입 수량 | 1주 (추후 변경 가능하도록 config로 관리) |

### 청산 조건 (OR, 먼저 도달하는 것)

| 조건 | 기준 |
|---|---|
| 트레일링 스탑 | 매입가 대비 +3% 돌파 후 최고점 대비 -2.5% 하락 시 청산 |
| 강제 익절 | 매입가 대비 +15% 도달 시 즉시 청산 |
| 손절 | 매입가 대비 -5% 도달 시 즉시 청산 |
| 타임컷 | 매입 후 30분 경과 + 손실 상태일 경우 청산 |
| 강제 청산 | 장 마감 10~5분 전 보유 전 종목 일괄 시장가 청산 |

***

## 청산 로직 흐름

```
매입 직후
  └─ 매입가 대비 수익률 실시간 감시

  +3% 미만 구간
    └─ 트레일링 없음
    └─ 손절(-5%), 타임컷(30분+손실), 강제 청산만 감시

  +3% 돌파 시
    └─ 트레일링 스탑 감시 시작
    └─ 최고점 갱신 추적
    └─ 최고점 대비 -2.5% 하락 시 청산

  +15% 도달 시
    └─ 트레일링 감시 중이어도 무관하게 즉시 청산
```

***

## 재진입 쿨다운

청산 사유(익절/손절/타임컷 무관)와 관계없이, 청산된 종목은 **청산 시점 기준 30분간 재진입 금지**.  
30분 경과 후 매입 조건을 재충족하면 진입 가능.

## 프로젝트 구조

```
hantu-kr-bot/
│
├── .env                  # API Key, 계좌번호 등 민감 정보 격리
├── config.json           # 매매 파라미터 (손익절 비율, 쿨다운 시간 등)
├── requirements.txt      # 파이썬 의존성 패키지
│
├── main.py               # 봇 메인 실행 진입점 및 비동기 루프 관리
├── kis_api.py            # KIS API 인증, WebSocket 시세, 주문 실행
├── market_monitor.py     # 거래량 폭등 종목 조회 및 30분 상승률 계산
├── position_manager.py   # 포지션 보유 현황, 트레일링 스탑, 청산 조건 감시
├── order_executor.py     # 매수/매도 주문 실행 및 체결 확인
├── scheduler.py          # 장 시작/마감 타임라인 감시, 강제 청산 실행
└── logger.py             # 거래 기록 및 로그 관리
```

***

## config.json 구조 (예시)

```json
{
  "entry": {
    "volume_surge": true,
    "price_rise_pct": 2.0,
    "price_rise_minutes": 30,
    "qty_per_trade": 1
  },
  "exit": {
    "trailing_trigger_pct": 3.0,
    "trailing_drop_pct": 2.5,
    "force_profit_pct": 15.0,
    "stop_loss_pct": -5.0,
    "timecut_minutes": 30,
    "force_close_minutes_before_market": 10
  },
  "cooldown_minutes": 30,
  "paper_trading": true
}
```

***

## 환경 설정

```bash
pip install -r requirements.txt
cp .env.example .env
# .env에 KIS API Key, 계좌번호 입력 후 실행
python main.py
```

***

## 주의사항

- 실제 주문 전 반드시 `paper_trading: true` 모드로 충분히 검증할 것.
- 봇 전용 서브 계좌를 별도로 개설하여 메인 자산과 격리 운영을 강력히 권장.
- 이 봇은 수익을 보장하지 않으며, 모든 투자 손실의 책임은 사용자에게 있음.
